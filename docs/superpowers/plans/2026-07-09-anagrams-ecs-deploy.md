# Anagrams ECS Fargate Deployment Scaffold — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a working, `terraform plan`-verified Terraform configuration that stands up an ECS Fargate deployment of the `anagrams-2` API (ECR, VPC/networking, RDS, Secrets Manager, IAM, ECS cluster/task/service, ALB), plus a production-appropriate Dockerfile for the API — matching the design already agreed in conversation and documented in the "Deploy container to AWS (ECS Fargate or Lambda)" Notion page.

**Architecture:** Public-subnet Fargate task (no NAT Gateway) behind an Application Load Balancer, RDS MySQL for the database, Secrets Manager for credentials, one Terraform root module under `docker-101/anagrams-2/terraform/`. Client (`am-client`) deployment via S3/CloudFront is a separate, later effort — out of scope here.

**Tech Stack:** Terraform (AWS provider ~> 5.0, random provider ~> 3.6), Docker, AWS ECS Fargate, ALB, RDS MySQL, Secrets Manager, ECR.

## Global Constraints

- Region: `us-east-1` everywhere (kept consistent for a later S3/CloudFront client deploy, which requires ACM certs in `us-east-1`).
- No NAT Gateway. Fargate tasks and RDS live in public subnets; access is locked down via security groups only (ALB SG → task SG → RDS SG chain). This was an explicit cost decision (NAT Gateway is ~$32/mo for a low-traffic demo).
- Fargate task sizing: 256 CPU units (0.25 vCPU) / 512 MB memory.
- Secrets (`DB_PASSWORD`, `JWT_SECRET`) are generated by Terraform (`random_password`) and stored only in Secrets Manager — never in `.tfvars`, plaintext `environment` blocks, or version control.
- Must match the existing app's real configuration exactly: container port `3001`, health check path `/health` (confirmed in `docker-101/anagrams-2/server/src/index.js:25` and `:421`), env var names `DB_HOST`/`DB_USER`/`DB_PASSWORD`/`DB_NAME`/`JWT_SECRET`/`JWT_EXPIRES_IN`/`CLIENT_URL` (confirmed in `docker-101/anagrams-2/server/src/index.js` and `docker-101/anagrams-2/server/.env.example`).
- All Terraform code lives under `docker-101/anagrams-2/terraform/`.
- **Scope stops at `terraform plan`.** No task in this plan runs `terraform apply` — that provisions real, billable AWS resources in the user's live account and should be a deliberate, watched action, not something done autonomously mid-plan. Each task's "test" is a real `terraform plan` against the user's live AWS account (credentials are already configured in this environment — `aws sts get-caller-identity` resolves to account `798294347421`), which is read-only and free, but gives a genuine correctness signal (not just offline syntax checking). The final task's README documents the exact `terraform apply` / `terraform destroy` commands for the user to run themselves.
- `server/Dockerfile.dev` (used for local dev with `nodemon` + volume mounts) is untouched. A separate production `server/Dockerfile` is created in Task 1 for what actually gets pushed to ECR.

- [ ] **Step 1: Write `network.tf`**

```hcl
resource "aws_vpc" "main" {
  cidr_block           = var.vpc_cidr
  enable_dns_support   = true
  enable_dns_hostnames = true

  tags = {
    Name = "${var.project_name}-vpc"
  }
}

resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id

  tags = {
    Name = "${var.project_name}-igw"
  }
}

resource "aws_subnet" "public" {
  count                   = length(var.public_subnet_cidrs)
  vpc_id                  = aws_vpc.main.id
  cidr_block              = var.public_subnet_cidrs[count.index]
  availability_zone       = var.availability_zones[count.index]
  map_public_ip_on_launch = true

  tags = {
    Name = "${var.project_name}-public-${var.availability_zones[count.index]}"
  }
}

resource "aws_route_table" "public" {
  vpc_id = aws_vpc.main.id

  route {
    cidr_block = "0.0.0.0/0"
    gateway_id = aws_internet_gateway.main.id
  }

  tags = {
    Name = "${var.project_name}-public-rt"
  }
}

resource "aws_route_table_association" "public" {
  count          = length(aws_subnet.public)
  subnet_id      = aws_subnet.public[count.index].id
  route_table_id = aws_route_table.public.id
}

resource "aws_security_group" "alb" {
  name        = "${var.project_name}-alb-sg"
  description = "Allow inbound HTTP from the internet"
  vpc_id      = aws_vpc.main.id

  ingress {
    description = "HTTP from internet"
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = {
    Name = "${var.project_name}-alb-sg"
  }
}

resource "aws_security_group" "ecs_task" {
  name        = "${var.project_name}-ecs-task-sg"

  ingress {
    description     = "App port from ALB"
    from_port       = var.container_port
    to_port         = var.container_port
    protocol        = "tcp"
    security_groups = [aws_security_group.alb.id]

  egress {
    from_port   = 0

    protocol    = "-1"

    Name = "${var.project_name}-ecs-task-sg"
```

```hcl
resource "aws_ecs_cluster" "main" {
  name = "${var.project_name}-cluster"

  setting {
    name  = "containerInsights"
    value = "disabled"
  }
}

resource "aws_cloudwatch_log_group" "api" {
  name              = "/ecs/${var.project_name}-api"
  retention_in_days = 14
}

resource "aws_ecs_task_definition" "api" {
  family                   = "${var.project_name}-api"
  requires_compatibilities = ["FARGATE"]
  network_mode             = "awsvpc"
  cpu                      = var.task_cpu
  memory                   = var.task_memory
  execution_role_arn       = aws_iam_role.ecs_execution.arn
  task_role_arn            = aws_iam_role.ecs_task.arn

  container_definitions = jsonencode([
    {
      name      = "api"
      image     = "${aws_ecr_repository.api.repository_url}:latest"
      essential = true

      portMappings = [
        {
          containerPort = var.container_port
          protocol      = "tcp"
        }
      ]

      environment = [
        { name = "DB_HOST", value = aws_db_instance.main.address },
        { name = "DB_USER", value = var.db_username },

        { name = "JWT_EXPIRES_IN", value = var.jwt_expires_in },
        { name = "CLIENT_URL", value = var.client_url }
      ]

      secrets = [
        {
          name      = "DB_PASSWORD"
          valueFrom = "${aws_secretsmanager_secret.app.arn}:DB_PASSWORD::"
        },
        {
          name      = "JWT_SECRET"
          valueFrom = "${aws_secretsmanager_secret.app.arn}:JWT_SECRET::"
        }
      ]

      logConfiguration = {
        logDriver = "awslogs"
        options = {
          "awslogs-group"         = aws_cloudwatch_log_group.api.name
          "awslogs-region"        = var.aws_region
          "awslogs-stream-prefix" = "api"

    }

  tags = {
    Name = "${var.project_name}-api-task"
```

## Cost notes

- The ALB has a flat hourly charge (~$16–20/mo) even at zero traffic — the
  biggest fixed cost here.
- RDS `db.t3.micro` is free-tier eligible (750 hrs/mo) on a new AWS account;
  otherwise ~$12–15/mo.
- Fargate task (256 CPU / 512 MB) is ~$9–13/mo if left running continuously.
- No NAT Gateway is provisioned (public-subnet + security-group pattern),
  which avoids its ~$32/mo cost.
